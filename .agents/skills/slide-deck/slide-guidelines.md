Training Slide Creation Rule Guide

1. Core Learning Principle

All training content should follow this learning cycle:

Problem
↓
Discussion
↓
Concept
↓
Example
↓
Takeaway

This structure ensures participants: 1. See the problem first 2. Think and discuss 3. Learn the concept 4. Understand through examples 5. Remember the key message

The goal is to create curiosity before explanation.

⸻

2. Training Deck Overall Structure

For long training sessions (e.g. 3–4 hours), organize the deck into modules.

Example structure:

Introduction
Module 1
Module 2
Module 3
Wrap-up

Each module contains multiple learning cycles.

Example:

Module 1
Cycle 1
Cycle 2
Cycle 3
Cycle 4

Typical slide estimate:

Section Slides
Introduction 5–10
Each learning cycle 5–10
Each module 25–40
4-hour training ~100–140 slides

⸻

3. Learning Cycle Implementation

Each learning cycle should follow this pattern.

Step 1 — Problem

Purpose: Trigger curiosity and relevance

Slides: 1

Examples:
• scenario
• question
• challenge
• real-world situation

Example slide:

A Shopify store receives 50
"Where is my order?" tickets every day.

Why does this happen?

Rules:
• Keep text minimal
• Present a realistic problem
• Encourage participants to think

⸻

Step 2 — Discussion

Purpose: Engage participants before teaching

Slides: 0–1

Example:

Discuss with a partner:

What could cause this problem?

Rules:
• Allow 1–3 minutes
• Ask open questions
• Encourage participation

⸻

Step 3 — Concept

Purpose: Introduce the framework or theory

Slides: 3–6

Rules:
• Explain step-by-step
• Avoid overwhelming information
• Focus on clarity

Example structure:

Slide 1 — Overview
Slide 2 — Concept part 1
Slide 3 — Concept part 2
Slide 4 — Concept part 3

Guideline:

1 idea = 1 slide

⸻

Step 4 — Example

Purpose: Show real-world application

Slides: 1–2

Example content:
• real case study
• screenshot
• chart
• comparison

Example:

Example: Poor tracking page UX

Rules:
• Prefer visual explanation
• Show before vs after
• Use real scenarios when possible

⸻

Step 5 — Takeaway

Purpose: Reinforce the key message

Slides: 1

Example:

Key takeaway:

Most WISMO issues are caused by
communication gaps, not shipping speed.

Rules:
• Summarize the concept
• Keep wording simple
• Highlight the key lesson

⸻

4. Slide Design Rules

Rule 1 — One Idea Per Slide

Each slide should communicate one clear message.

Avoid combining multiple concepts in one slide.

Bad:

5 causes of WISMO

- No tracking
- Delivery delay
- Inventory issues
- Address errors
- Carrier issues

Better:

Cause #1
No shipping notification

⸻

Rule 2 — Visual First

Prefer visuals over text.

Slide structure:

Headline
Visual (image / chart / diagram)
Optional short explanation

Example:

Customers hate shipping uncertainty

[tracking page screenshot]

⸻

Rule 3 — Short Headlines

Use clear and concise headlines.

Recommended length:

≤ 6–8 words

Examples:

Good:

Why customers ask "Where is my order"

Better:

Customers hate shipping uncertainty

⸻

Rule 4 — Reveal Information Gradually

Avoid showing too much information at once.

Instead of:

3 causes of WISMO

Reveal each point separately:

Cause #1
Cause #2
Cause #3

This maintains audience attention.

⸻

Rule 5 — Slides Are Not Documents

Slides should support your explanation, not replace it.

Avoid:
• paragraphs
• long bullet lists
• dense text blocks

Slides should feel incomplete without the presenter.

⸻

5. Cognitive Load Management

To maintain attention during long sessions:

Change activity every ~10 minutes

Pattern:

Problem
Discussion
Concept
Example
Takeaway

Insert engagement moments

Examples:

What surprised you the most?

Have you experienced this problem?

Which option would you choose?

This resets audience focus.

⸻

6. Recommended Slide Content Ratio

For effective training decks:

Type Percentage
Visual slides 60–70%
Charts / diagrams 10–15%
Text explanation 15–20%

This keeps slides engaging and easy to understand.

⸻

7. Slide Creation Workflow

When turning ideas into slides, follow this process.

Step 1 — Identify the learning objective

What should participants understand after this section?

Step 2 — Define the problem

What real situation illustrates this concept?

Step 3 — Build the cycle

Problem
Discussion
Concept
Example
Takeaway

Step 4 — Expand the concept section

Break complex ideas into multiple slides.

Step 5 — Add visuals

Use diagrams, screenshots, or charts.

Step 6 — Refine slides

Apply design rules:
• one idea per slide
• minimal text
• clear headline

⸻

8. Quick Checklist Before Finalizing Slides

Check the following:
• Does each cycle start with a problem?
• Are participants given a chance to think or discuss?
• Is each concept explained step-by-step?
• Are real examples included?
• Does each section end with a clear takeaway?
• Does every slide communicate one idea?

If all answers are yes, the training deck is well structured.
